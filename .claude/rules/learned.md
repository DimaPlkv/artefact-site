# Learned — паттерны и исправления

### 2026-06-14 — Tilda DNS не поддерживает CNAME — нужно переносить NS на регистратора
Tilda позволяет добавлять только TXT и MX через свой UI. Для CNAME (например, поддомен → GitHub Pages) нужно в Reg.ru сменить NS-серверы с `ns1.tildadns.com` на `ns1.reg.ru`, после чего в Reg.ru появляется полный редактор DNS-зоны. Основной сайт на Тильде продолжает работать — A-записи (@ и www → Tilda IP) остаются в зоне. Смена NS распространяется 1–4 часа.

### 2026-06-14 — Flex-колонка без определённой высоты родителя: flex:1 даёт 0
`.card-promo` на desktop: `grid-row: span 2` задаёт высоту через грид → `flex: 1` на фото работает. На мобайле `grid-row: span 1` с одним элементом в ряду не даёт высоту → фото схлопывается в 0. **Решение:** `display: block` + явный `aspect-ratio` на фото для мобайла. Паттерн применим к любому flex-дочернему элементу без явной высоты родителя.

### 2026-06-14 — admin.html DEFAULT-данные и index.html могут расходиться незаметно
DEFAULT в admin.html используется только когда localStorage пуст — при реальной работе данные идут из localStorage. Из-за этого расхождения (разные имена полей, разный регистр значений, разная структура) не всплывают сразу. **Паттерн аудита:** сравнивать: (1) имена полей в render/save функциях admin vs SITE_DATA в index.html; (2) значения статусов через `===` в index.html vs `data-val` в admin.html; (3) что saveXxx() реально пишет в data vs что index.html читает. **Конкретные баги:** `about.stats` vs `about.facts`, статусы `'в городе'`/`'уничтожен'` vs `'В городе'`/`'Уничтожена'`, форма автора не сохраняла `technique`/`hometown`/`yearJoined`/`quote`/`quoteHl`.

### 2026-05-29 — opacity на дочернем элементе не работает если родитель opacity:0
Если родитель имеет `opacity: 0`, дочерний элемент нельзя сделать видимым через `opacity: 1` — дочерние opacity перемножаются. **Решение:** выносить элемент из прозрачного родителя и позиционировать абсолютно напрямую в контейнер.

### 2026-05-29 — justify-content: space-between с одним дочерним элементом
При одном child в flex-контейнере с `space-between` элемент прижимается к началу (как `flex-start`), а не к концу. После удаления второго child нужно явно поменять на `flex-end`. 

### 2026-05-29 — src="" на img триггерит onerror
`<img src="">` немедленно вызывает `onerror` в браузере. Нельзя вешать `onerror="..."` на статический img с пустым src — класс-заглушка навешивается сразу и не снимается. **Решение:** добавлять `onerror` программно в JS одновременно с установкой реального `src`, и сбрасывать `classList.remove('img-error')` перед каждой сменой src.

### 2026-05-28 — sips: правильный флаг для ресайза — `-Z`, не `--resampleLongSide`
`sips -Z 1600` ограничивает длинную сторону до 1600px. `--resampleLongSide` не существует в macOS sips. Полная команда: `sips -s format jpeg -s formatOptions 75 -Z 1600 input.png --out output.jpg`

### 2026-05-21 — JS-параллакс конфликтует с CSS-анимациями на hero-слоях
Параллакс пишет `style.transform` на `.hero-layer` каждый кадр. CSS-анимация на том же элементе перезаписывается и не работает. **Решение:** анимировать inner-элементы (`.hero-artefact-svg`, `.hero-muzey-svg`), а не `.hero-layer`.

### 2026-05-21 — applySettings() перезаписывал SVG-логотип
`else { lm.textContent = s.logoMain }` — срабатывал при каждой загрузке, заменяя инлайн-SVG текстом. **Решение:** убрать else-ветку полностью; применять только если `s.logoImage` задан явно.

### 2026-05-21 — align-items: start обязателен для грида с карточками разной высоты
Grid по умолчанию растягивает items до высоты строки (`stretch`). Карточки с длинными и короткими заголовками выглядят разного размера. **Решение:** `align-items: start` + `min-width: 0` на карточках внутри грида.

### 2026-05-21 — sips повторная компрессия JPEG малоэффективна
Пережатие уже сжатого JPEG через `sips -s formatOptions 75` даёт минимальный выигрыш. Детальные уличные фото (камень, асфальт) при 1600px весят 300–800kb — это норма. Не пытаться сжимать ниже.

### 2026-05-22 — @media override не работает если стоит ДО базового правила
Мобильный `.hero { height: 520px }` в `@media (max-width: 768px)` стоял до базового `.hero { height: 100vh }`. CSS читается сверху вниз: базовое правило побеждало на мобиле. **Решение:** mobile overrides ставить ПОСЛЕ всех базовых правил.

### 2026-05-22 — localStorage в index.html перезаписывал SITE_DATA
После `SITE_DATA_END` блок читал из localStorage и перезаписывал данные файла. Правки `index.html` игнорировались даже после hard refresh. **Решение:** блок удалён. `index.html` теперь только ПИШЕТ в localStorage, не читает.

### 2026-05-22 — JSON.parse не парсит JS-объектные литералы
`SITE_DATA` использует JS-синтаксис: одинарные кавычки, ключи без кавычек. `JSON.parse()` падает с SyntaxError. **Решение:** `new Function('return (' + jsStr + ')')()`.

### 2026-05-22 — Синк admin→GitHub перетирал локальные данные
Автосинк admin тянул SITE_DATA с GitHub и писал в localStorage, уничтожая локальные правки. **Правильная архитектура:** index.html → localStorage → admin. GitHub только при публикации.

### 2026-05-22 — Единый aspect-ratio 4:3 для всех фото-контекстов
Галерея, слайдер экспоната, фото автора — все `aspect-ratio: 4/3`. Обложка в 4:3 выглядит одинаково везде.

### 2026-05-23 — MapLibre popup: жёсткая ширина inner-div конфликтует с padding
`width:220px` на inner-div, но контентная зона = `min-width(260px) - padding(24px×2) = 212px`. Изображение `width:100%` вылезало за край. **Решение:** `width:100%` на div, padding одинаковый (`16px` со всех сторон).

### 2026-05-23 — Negative margin image-bleed в MapLibre popup ненадёжен
`margin: -15px -15px 10px -15px` не работает без точного знания padding попапа. **Решение:** равные padding + `width:100%`. Для bleed — переопределить `.maplibregl-popup-content { padding: 0 }` и добавить padding вручную к тексту.

### 2026-06-05 — display:block на родителе убивает ::before с width/height
`::before` по умолчанию `display:inline`. Если родитель был `display:flex` (что делает ::before flex-item с работающими width/height), а затем переопределён на `display:block` — pseudo-element становится inline и width/height перестают работать. Точка-индикатор исчезает. **Решение:** не переопределять `display` у элементов с ::before-dot. Для отступа использовать только `margin-top`, без смены display.

### 2026-06-05 — @media override не работает если стоит ДО базового (повтор)
`.map-sidebar` — мобильный `@media (max-width:960px)` с `width:min(360px,...)` стоял ДО базового `.map-sidebar { width: calc(50% - var(--edge)) }`. Базовое правило побеждало на мобайле. **Решение:** десктопную ширину обернуть в `@media (min-width:961px)`, чтобы не конкурировать с мобильным override.

### 2026-06-05 — white-space: pre-line для многострочного textContent
Абзацы в строке через `\n\n` + `element.textContent = str` рендерятся как один абзац без `white-space: pre-line`. Паттерн уже использовался для `logoSub` — применим ко всем полям с многострочным текстом (bio автора и т.д.). **Решение:** добавить `white-space: pre-line` в CSS и хранить разрывы как `\n\n` в SITE_DATA.

### 2026-06-05 — MapLibre flyTo popup уходит за экран без padding
`flyTo({ center })` центрирует маркер строго посередине карты. Попап открывается вниз и вылезает за край. **Решение:** `flyTo({ padding: { top: 40, bottom: 360, left: 40, right: 40 } })` — маркер смещается в верхнюю треть, попап раскрывается в пределах экрана.

### 2026-06-08 — Алгоритм добавления фото экспоната/автора
1. Конвертировать PNG/HEIC → JPEG (sips, quality 75, Z 1600) в Latin-папку назначения
2. Удалить исходную папку пользователя (`rm -r`) — **всегда, не ждать просьбы**
3. Прописать пути в SITE_DATA
Если папка с Кириллицей — переименовать в Latin transliteration (строчные, дефис).

### 2026-06-08 — Inline hover-кнопка со своим padding сдвигает baseline
`padding: 3px 7px` на `display: inline-flex` кнопке визуально опускает текст относительно соседних ячеек грида/флекса. **Решение:** компенсировать через `margin: -3px 0 -3px -7px` — отрицательный margin равен padding. Всегда проверять baseline соседних элементов при изменении padding на inline-кнопке.

### 2026-06-12 — macOS HFS+ регистронезависима: mkdir lowercase не создаёт новую папку если uppercase уже есть
`mkdir nevinosimaya` при существующей `Nevinosimaya` — та же папка. sips пишет туда же, потом `rm -r Nevinosimaya` сносит JPEGи вместе с оригиналами. **Решение:** конвертировать в `/tmp/name-out/`, удалить источник, потом `mv /tmp/name-out "$BASE/name"`.

### 2026-05-23 — MapLibre: язык подписей меняется только после события load
OpenFreeMap positron грузит подписи на английском. Переключить на русский: в обработчике `load` итерировать symbol-слои и вызывать `map.setLayoutProperty(id, 'text-field', ['coalesce', ['get', 'name:ru'], ['get', 'name']])`. Для каждой карты отдельно.
