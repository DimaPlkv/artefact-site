# /analyze — анализ ошибок и самообучение

Запускай после каждого значимого блока работы. Обязательно, не опционально.

## Что делать

1. Просмотри все изменения этого блока работы
2. Для каждой нетривиальной ошибки или находки:
   - Что пошло не так / что было неожиданным?
   - Почему это произошло (неверное предположение, конфликт механизмов)?
   - Какое правило это порождает на будущее?
3. Обнови `.claude/rules/learned.md` — добавь новые записи
4. Обнови соответствующий раздел CLAUDE.md и `.claude/commands/site.md` если архитектура или стек изменились
5. Обнови memory-файлы в ~/.claude/projects/.../memory/ если есть что запомнить между сессиями

## Критерии для записи в Learned

**Записывать:**
- Баг который я уже видел раньше или мог предвидеть по архитектуре
- Конфликт между двумя механизмами (CSS каскад, JS порядок выполнения, etc.)
- Неверное предположение о поведении браузера / API / инструмента
- Паттерн который сэкономит время в следующий раз

**Не записывать:**
- Опечатки
- Добавление контента
- Рутинные CSS-правки

## Формат записи

```
### YYYY-MM-DD — Короткий заголовок
Что произошло. **Решение:** что сделали.
```
