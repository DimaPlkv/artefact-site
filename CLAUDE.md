# Артефакт — музей размером с город

Сайт-проект: городской музей, где экспонаты расположены по всему Томску (и другим городам).

## Файлы

- `index.html` — публичный сайт. Весь HTML, CSS и JS в одном файле.
- `admin.html` — панель управления контентом. Тоже self-contained.
- `Images/` — фоновые изображения для hero-секции (Sky.png, City.png).
- `photos/` — реальные фото экспонатов, организованы по папкам (`photos/название-экспоната/`).

## Архитектура данных

Все данные сайта хранятся прямо внутри `index.html` между маркерами:
```
/* __SITE_DATA_START__ */
const SITE_DATA = { ... };
/* __SITE_DATA_END__ */
```

`SITE_DATA` содержит: `artifacts[]`, `about`, `manifest`, `contacts`, `authors[]`, `settings`.

**Синк index.html ↔ admin.html:**
- `index.html` при каждой загрузке пишет `SITE_DATA` в `localStorage('artefact-site')`
- `admin.html` читает из `localStorage` и редактирует там
- При публикации admin берёт локальный `index.html` как шаблон, вставляет новый `SITE_DATA`, пушит на GitHub
- Итог: открыл `index.html` → обновился localStorage → admin видит актуальные данные автоматически

## Структура index.html

Навигация между разделами через функцию `navigate(view, id)`:
- `view-main` — галерея + hero-блок
- `view-about` — о музее (факты + манифест)
- `view-artifact` — карточка экспоната (фотослайдер + мини-карта)
- `view-contacts` — контакты
- `view-authors` — список авторов
- `view-author` — страница автора
- `view-map` — общая карта всех экспонатов

`navigateToGallery()` — отдельная функция, якорный скролл к gallery-section минуя hero.

## Стек

- Чистый HTML/CSS/JS, никаких фреймворков и бандлеров
- **MapLibre GL JS 4** для карт (OpenFreeMap positron, без API-токена)
- Шрифт Inter (Google Fonts)
- Всё в одном файле — легко деплоить как статику

## Стиль кода

- CSS-переменные в `:root` (`--black`, `--white`, `--border`, `--edge`, `--nav-h`)
- Функции в глобальном скоупе
- Комментарии-разделители вида `/* ── SECTION ──────── */`

## Важные технические детали

- **MapLibre координаты**: порядок `[lng, lat]` — не путать с Leaflet (`[lat, lng]`)
- **Hero-анимация**: JS-параллакс владеет `style.transform` на `.hero-layer`. CSS-анимации применяются к inner-SVG элементам (`.hero-artefact-svg` и т.д.), иначе конфликт
- **Логотип**: SVG встроен инлайн в `.nav-logo-main` с `fill="currentColor"`. `applySettings()` не должен перезаписывать его текстом — только если загружена кастомная картинка
- **logoSub**: хранится с `\n`, рендерится через `white-space: pre-line` + `ls.textContent = s.logoSub`
- **Автор неизвестен**: не добавляется в массив `authors[]` — только опция в select админки. На странице экспоната отображается как plain text без ссылки (стандартное поведение когда `authorObj` не найден)
- **Фото**: HEIC конвертируется через `sips`, качество 75, максимум 1600px по длинной стороне. Путь: `photos/название-папки/IMG_XXXX.jpg`

## Протокол самообучения — ОБЯЗАТЕЛЬНО

После каждого блока работы (не только по напоминанию):
1. Были ли нетривиальные ошибки или неожиданные находки?
2. Если да — сразу добавить в `## Learned` ниже
3. Если изменилась архитектура/стек — обновить соответствующий раздел
4. Использовать `/analyze` для структурированного разбора

**Записывать в Learned:**
- Баг из-за неверного предположения о поведении браузера/CSS/JS
- Конфликт между двумя механизмами (CSS каскад, порядок JS, etc.)
- Паттерн который сэкономит время в следующий раз

**Не записывать:** опечатки, добавление контента, рутинные правки CSS

## Стратегия сабагентов

Использовать `Agent` когда задача требует широкого чтения или изолированного контекста:

| Когда | Тип агента |
|-------|-----------|
| Аудит синка admin↔index (читать 2 больших файла) | `general-purpose` |
| Конвертация/оптимизация пачки фото | `general-purpose` |
| Проверка безопасности всего JS | `general-purpose` |
| Поиск символа или паттерна в коде | `Explore` |

Простые правки CSS/текста/одного блока — делать напрямую, без сабагентов.

## Learned

_Паттерны и исправления, накопленные в процессе работы._

### 2026-05-21 — JS-параллакс конфликтует с CSS-анимациями на hero-слоях
Параллакс пишет `style.transform` на `.hero-layer` каждый кадр. CSS-анимация на том же элементе перезаписывается и не работает. **Решение:** анимировать inner-элементы (`.hero-artefact-svg`, `.hero-muzey-svg`), а не `.hero-layer`.

### 2026-05-21 — applySettings() перезаписывал SVG-логотип
`else { lm.textContent = s.logoMain }` — срабатывал при каждой загрузке, заменяя инлайн-SVG текстом. **Решение:** убрать else-ветку полностью; применять только если `s.logoImage` задан явно.

### 2026-05-21 — align-items: start обязателен для грида с карточками разной высоты
Grid по умолчанию растягивает items до высоты строки (`stretch`). Карточки с длинными и короткими заголовками выглядят разного размера. **Решение:** `align-items: start` + `min-width: 0` на карточках внутри грида.

### 2026-05-21 — sips повторная компрессия JPEG малоэффективна
Пережатие уже сжатого JPEG через `sips -s formatOptions 75` даёт минимальный выигрыш. Детальные уличные фото (камень, асфальт) при 1600px весят 300–800kb — это норма. Не пытаться сжимать ниже.

### 2026-05-22 — @media override не работает если стоит ДО базового правила
Мобильный `.hero { height: 520px }` в `@media (max-width: 768px)` стоял на строке 137, а базовый `.hero { height: 100vh }` — на строке 232. CSS читается сверху вниз: базовое правило побеждало даже на мобиле. **Решение:** mobile overrides для hero перенесены в `@media (max-width: 540px)` который стоит после всех базовых правил.

### 2026-05-22 — localStorage в index.html перезаписывал SITE_DATA, скрывая изменения файла
После `SITE_DATA_END` был блок `(function(){ const d = localStorage.getItem('artefact-site'); if(d) SITE_DATA.artifacts = d.artifacts... })()` — он читал из localStorage и перезаписывал данные файла. Любые правки `index.html` игнорировались браузером даже после hard refresh. **Решение:** блок удалён. Направление изменено: `index.html` теперь ПИШЕТ в localStorage (`localStorage.setItem('artefact-site', JSON.stringify(SITE_DATA))`), а не читает.

### 2026-05-22 — JSON.parse не парсит JS-объектные литералы
`SITE_DATA` использует JS-синтаксис: одинарные кавычки, ключи без кавычек. `JSON.parse()` на этом падает с SyntaxError. **Решение:** `new Function('return (' + jsStr + ')')()` — корректно вычисляет JS-выражение.

### 2026-05-22 — Синк admin→GitHub перетирал локальные данные
Автосинк admin при открытии тянул SITE_DATA с GitHub и писал в localStorage. Если на GitHub была старая версия (пользователь удалял экспонаты) — локальные правки из index.html терялись. **Правильная архитектура:** index.html → localStorage → admin (а не GitHub → localStorage). Admin тянет с GitHub только по явному запросу при публикации.

### 2026-05-22 — Единый aspect-ratio 4:3 для всех фото-контекстов
Галерея (`.card-photo`), слайдер экспоната (`.artifact-photo-col`), фото автора (`.author-photo-col`) — все переведены на `aspect-ratio: 4/3`. Обложка подготовленная в 4:3 выглядит одинаково везде. Мобильные overrides тоже обновлены.
